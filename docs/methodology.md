# Methodology

SotA Lens operationalizes an exploratory review workflow:

1. define seed fields and search terms;
2. collect seed records from a scholarly search source;
3. enrich records with metadata and references when possible;
4. build a directed citation graph;
5. inspect graph structure, communities, ranked authors, and subject terms;
6. use the resulting map to refine review questions, search terms, and inclusion/exclusion criteria.

The method is designed for early-stage state-of-the-art and scoping work. It is not a replacement for systematic screening, critical appraisal, or PRISMA-style reporting.

## Complementarity with formal review protocols

Formal protocols are strongest when the review question and screening procedure are already stable. SotA Lens is useful before that point, when the aim is to understand how a field is organized and which neighbouring areas should be included.

Typical outputs include:

- a transparent seed corpus;
- a citation network;
- clusters or components that suggest subfields;
- candidate terms for refined searches;
- influential authors, institutions, papers, and review articles;
- anomalies and missing areas requiring manual checking.

## Limitations

Citation graphs reflect the data source and metadata quality. Missing DOIs, incomplete references, ambiguous author names, and inaccessible records can distort the graph. Graph communities should be treated as interpretive aids rather than objective topic labels. Final claims should always be checked against manual reading and explicit screening criteria.
